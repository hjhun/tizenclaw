# tizenclaw-cli 개선 피드백 수집 질의

재사용 가능한 입력 프롬프트 파일입니다.
아래 사용 방법에 따라 `tizenclaw-cli`에 전달하여 개선 요구사항을 수집합니다.

## 사용 방법

```bash
# 1. 데몬 실행 여부 확인
./deploy_host.sh --status

# 2. 데몬이 없으면 시작
./deploy_host.sh

# 3. 아래 질의 텍스트를 tizenclaw-cli로 전달
tizenclaw-cli "$(cat tizenclaw-cli-피드백-질의.md | sed -n '/^## 질의 텍스트 \(전달용\)$/,/^---/p' | sed '1d;/^---/d')"
```

또는 질의 텍스트 섹션을 복사해서 직접 붙여넣기:

```bash
tizenclaw-cli "..."
```

## 질의 텍스트 (전달용)

tizenclaw-cli를 사용하는 관점에서 아래 네 가지를 구체적으로 정리해줘.

1. 현재 무엇이 부족한가: 지금 당장 쓰기 불편하거나 없어서 아쉬운 기능을 모두 나열해줘. UX, 성능, 기능 누락, 에러 처리, 운영성 등 모든 측면을 포함해줘.

2. 무엇이 약하거나 불편한가: 기능은 있지만 동작 방식이 불만족스럽거나 개선 여지가 큰 부분을 설명해줘.

3. 무엇을 먼저 개선해야 하는가: 위 항목 중 가장 먼저 손봐야 할 것을 우선순위 순서로 정리해줘. 각 항목마다 왜 그 순서인지 이유도 함께 설명해줘.

4. 구체적인 예시: 원하는 명령어 형태, 워크플로우, 또는 응답 동작 방식의 구체적인 예시를 들어줘. 예를 들어 "이런 명령어가 있으면 좋겠다", "이런 상황에서 이렇게 동작하면 좋겠다" 같은 형태로 설명해줘.
