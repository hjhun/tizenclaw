# tizenclaw-cli 개선 요구사항 프롬프트

## 수집 방법

```
tizenclaw-cli "tizenclaw-cli를 사용하면서 부족하다고 느끼는 점과 개선하고 싶은 기능이
무엇인지 정리해줘. UX, 성능, 기능 누락, 에러 처리 등 모든 측면에서 현재 tizenclaw-cli의
문제점과 개선 방향을 구체적으로 설명해줘."
```

실행 일자: 2026-04-18  
소켓: `~/.tizenclaw/run/tizenclaw.sock`

## 에이전트 응답에서 도출한 개선 요구사항

### 우선순위 높음

1. **작업 진행 상태 표시 강화**  
   복합 작업 시 `입력 분석 중`, `에이전트 위임 중`, `결과 정리 중` 등 단계별 상태를
   실시간으로 출력하라. 장기 작업에는 spinner와 elapsed time을 표시하라.

2. **에러 메시지 구조화**  
   모든 에러 출력에 원인·영향·해결 방법·재시도 예시를 포함하라.  
   나쁜 예: `tool call failed`  
   좋은 예: `run_coding_agent 호출이 120초 내 완료되지 않았습니다. [원인] 백엔드 응답
   지연. [해결] --timeout-secs 300 옵션을 추가하세요.`

3. **작업 종료 시 읽기/쓰기/변경 파일 요약 출력**  
   AI CLI가 어떤 파일을 읽고 수정했는지 사용자가 알 수 있도록 완료 시 요약을 출력하라.

4. **dry-run / confirm-plan 모드 추가**  
   `--dry-run` 또는 `--confirm-plan` 플래그로 실제 실행 전에 어떤 도구와 에이전트를
   호출할지 미리 보여주는 모드를 제공하라.

5. **세션 관리 명령 추가**  
   `tizenclaw-cli session list`, `session resume <id>`, `session show <id>` 명령을 구현하라.

6. **JSON 출력 안정화**  
   `--json` 플래그가 CI/스크립트에서 신뢰할 수 있도록 스키마를 고정하고, exit code 체계를
   정교화하라. stdout/stderr 역할을 명확히 분리하라.

7. **대형 프로젝트 스캔 최적화**  
   `.gitignore`와 `.tizenclawignore`를 활용해 불필요한 전체 트리 재탐색을 방지하라.
   변경 파일 우선 분석(git diff 기반)을 기본 동작으로 채택하라.

8. **부분 실패 처리 개선**  
   복합 작업에서 일부만 실패할 때 성공한 단계와 실패한 단계를 분리해 출력하고, 실패한
   부분만 재시도하는 옵션을 제공하라.

### 우선순위 중간

- `tizenclaw-cli doctor` 명령으로 환경 진단 기능 제공
- `tizenclaw-cli ps` / `status` / `top`으로 리소스 상태 확인
- `tizenclaw-cli analyze project` 등 프로젝트 진단 전용 명령 추가
- `--debug`, `--trace` 레벨 분리 및 debug bundle export 기능
- 프로젝트별 `.tizenclaw.toml` 설정 파일 지원
- 협업용 `--format markdown` / `--format json` 출력 모드
- PR 설명문·커밋 메시지 자동 생성

## 핵심 진단 요약

| 관점       | 문제                                       |
|------------|---------------------------------------------|
| UX         | 진행 상태가 보이지 않고 사용자 제어권이 약함 |
| 성능       | 단순 작업에도 과한 탐색/위임 발생 가능       |
| 기능       | 세션 관리·검증 루프·프로젝트 분석 부족      |
| 에러 처리  | 원인보다 복구 방법이 부족                   |
| 신뢰성     | 읽기/변경 내역이 불투명해 감사 어려움        |
| 제품성     | 자동화 친화성·discoverability 부족          |
