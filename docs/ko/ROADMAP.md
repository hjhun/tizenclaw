## 개요

이 문서는 TizenClaw — Tizen OS용 임베디드 AI 에이전트 데몬의 개발 역사, 현재 상태, 향후 방향을 기술합니다. TizenClaw는 20개 개발 단계를 완료했으며, 현재 Phase 21을 진행 중입니다.

### 현재 상태: **Phase 21 — 프레임워크 안정화 및 SDK 내보내기** 🔴 진행 중

## 지원 기능 (✅ 구현 완료)

### 핵심 시스템
| 기능 | 설명 |
|------|------|
| **Agentic Loop** | LLM → 도구 호출 → 결과 → LLM 반복 루프 (설정 가능한 max_iterations) |
| **스트리밍 응답** | 청크 IPC 전달 + Telegram 점진적 편집 |
| **컨텍스트 압축** | 15턴 초과 시 LLM 기반 자동 요약 |
| **엣지 메모리 관리** | 유휴 시 `malloc_trim` + SQLite 캐시 플러시 |
| **멀티 세션** | 세션별 시스템 프롬프트 및 히스토리 격리 |

### LLM 백엔드
| 기능 | 설명 |
|------|------|
| **5개 내장 백엔드** | Gemini, OpenAI, Anthropic, xAI, Ollama |
| **통합 우선순위 전환** | Plugin > active > fallback 자동 폴백 |
| **RPK LLM 플러그인** | 재컴파일 없는 런타임 백엔드 확장 |
| **사용량 추적** | 세션별/일별/월별 Markdown 리포트 |

### 통신 채널
| 기능 | 설명 |
|------|------|
| **7개+ 채널** | Telegram, Slack, Discord, MCP, Webhook, Voice, Web Dashboard |
| **아웃바운드 메시징** | LLM 주도형 능동 알림 및 브로드캐스트 |
| **채널 플러그인** | `.so` 공유 라이브러리 플러그인 (C API) |

### 스킬 및 도구
| 기능 | 설명 |
|------|------|
| **13개 네이티브 CLI 스위트** | 35개+ 디바이스 API (배터리, Wi-Fi, BT, 디스플레이 등) |
| **20개+ 내장 도구** | 코드 실행, 태스크, 세션, RAG, 워크플로우, 파이프라인, 메모리 |
| **RPK 스킬 플러그인** | 플랫폼 서명된 Resource Package를 통한 Python 스킬 |
| **CLI 도구 플러그인** | `.tool.md` 설명서를 가진 네이티브 TPK 바이너리 |
| **Tizen Action Framework** | 자동 발견 및 이벤트 동기화를 가진 Per-action 타입 도구 |
| **핫리로드** | inotify 기반 스킬 감지 (데몬 재시작 불필요) |

### 보안
| 기능 | 설명 |
|------|------|
| **OCI 컨테이너 격리** | crun + PID/Mount/User 네임스페이스 + seccomp |
| **도구 실행 정책** | 위험도 (low/medium/high), 차단 스킬 목록 |
| **API 키 암호화** | 디바이스 바인딩 GLib SHA-256 + XOR |
| **감사 로깅** | 일별 Markdown 테이블, 5MB 로테이션 |
| **플랫폼 인증서 서명** | RPK/TPK 플러그인 설치 시 필수 |

### 지식 및 인텔리전스
| 기능 | 설명 |
|------|------|
| **하이브리드 RAG 검색** | BM25 키워드(FTS5) + 벡터 코사인, RRF(k=60) |
| **온디바이스 임베딩** | ONNX Runtime `all-MiniLM-L6-v2` (384차원, 지연 로드) |
| **영속 메모리** | 장기/에피소드/단기 + `remember`/`recall`/`forget` |

### 자동화 및 오케스트레이션
| 기능 | 설명 |
|------|------|
| **태스크 스케줄러** | Cron/interval/once/weekly + LLM 통합 + 재시도 |
| **슈퍼바이저 에이전트** | 목표 분해 → 역할 에이전트 위임 → 검증 |
| **스킬 파이프라인** | 변수 보간을 가진 결정적 순차 실행 |
| **자율 트리거** | LLM 평가를 가진 이벤트 기반 규칙 엔진 |
| **A2A 프로토콜** | 크로스 디바이스 에이전트 통신 (JSON-RPC) |

### 운영 및 관리
| 기능 | 설명 |
|------|------|
| **웹 대시보드** | 다크 글래스모피즘 SPA (포트 9090) |
| **헬스 모니터링** | Prometheus 스타일 `/api/metrics` + 실시간 대시보드 |
| **OTA 업데이트** | 버전 확인 및 롤백을 가진 OTA 스킬 업데이트 |
| **보안 터널링** | ngrok을 통한 원격 대시보드 접근 |

### TizenClaw 고유 강점

| 강점 | 설명 |
|------|------|
| **네이티브 C++ 성능** | ~812KB 바이너리, ~8.5MB idle PSS — 임베디드 디바이스에 최적 |
| **엣지 메모리 관리** | `malloc_trim` + SQLite 캐시 플러시를 통한 적극적 idle 회수 |
| **직접 Tizen C-API** | ctypes FFI + Tizen Action Framework를 통한 35개+ 디바이스 API |
| **OCI 컨테이너 격리** | crun 기반 seccomp + namespace — 세밀한 시스콜 제어 |
| **경량 배포** | systemd + RPM — Node.js/Docker 런타임 불필요 |
| **플러그인 생태계** | RPK 스킬, TPK CLI 도구, SO 채널, RPK LLM 백엔드 |
