Fix reviewer findings from rework pass 11

Address three issues found during Phase 5 review of the runtime
flexibility improvements:

1. Plugin-discovered backends were incorrectly dropped when `providers[]`
   was present. The old retain predicate removed any backend not listed in
   the enabled set; the fix uses `unwrap_or(true)` so plugin-discovered
   backends absent from `providers[]` remain as last-resort fallbacks.
   Applied to both startup and reload paths in runtime_core_impl.rs.

2. An explicit empty `providers: []` was not treated as authoritative
   because the sort/retain block was guarded by `!ordered_names.is_empty()`,
   skipping the block for an empty providers array. Added
   `providers_array_present` to `ProviderRoutingConfig` and changed the
   guard to also enter the block when the key is explicitly present.

3. The `backends.<name>.model` fallback in
   `read_backend_models_from_llm_config` was dead in practice for Gemini
   because the builtin default model caused `is_some()` to skip the
   fallback. Replaced the guard with a before/after snapshot comparison
   so the fallback only skips when the telegram section explicitly changed
   the model. Updated the test to remove the manual model-clearing
   workaround.
